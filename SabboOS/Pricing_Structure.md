## Part 1: The Two Product Lines

We sell two distinct things that share infrastructure:

1. **Coaching Programs** (Amazon OS — students get taught + tools)
2. **Standalone SaaS** (non-coaching Amazon sellers pay for just the AI software)

Both live on 247profits.org. Both use the same credit system under the hood. But the pricing logic is completely different — coaching is transformational ($ vs outcome), SaaS is transactional ($ vs tool usage).

## Part 4: Credit Economics — Why These Numbers

Real per-action cost (internal reference — don't expose to customers):

**Key infrastructure fact:** Nova Chat and all AI features route through **Claude CLI via Sabbo's Claude Max plan ($200/mo flat)**. AI inference has ZERO marginal cost as long as we stay under Max plan limits. We do NOT pay Anthropic API per-token for these features.

| Action | Keepa tokens | AI cost | Scraper compute | Our cost | Credits charged |
|---|---|---|---|---|---|
| Product Analyzer (one-shot) | 5 | $0 (Claude Max) | — | ~$0.02 | 5 credits |
| Seller scan (per seller) | 21 × N products | $0 | 0.02s | ~$0.10 | 10 credits |
| Nova Chat message | — | $0 (Claude Max) | — | ~$0 | 1 credit |
| Deal scan (50 products) | 50 | $0 | 5s | ~$0.20 | 5 credits |
| Gift Card Intel refresh | 0 (scraped) | 0 | 10s | ~$0.01 | 0 credits (free) |
| Wholesale Research query | — | $0 (Claude Max) | 30s | ~$0.02 | 15 credits |
| Listing Optimizer | 5 | $0 | — | ~$0.02 | 5 credits |

**Margin check:** The only real variable cost is Keepa tokens + scraper compute. At $29/mo Starter, a heavy user burning 1,500 credits costs us ~$15 in Keepa max → **50%+ margin even on Starter**. AI features are essentially free infra; Keepa is the bottleneck.

**Two backstops, not one:**
1. `max_tracked_sellers` — caps continuous Keepa scan load (this is the real Keepa-burner)
2. Claude Max plan usage limits — if Sabbo hits Max plan caps, throttle AI features org-wide (never let one user DOS the Claude Max plan for everyone)

**The backstop isn't credits, it's `max_tracked_sellers`** + Claude Max capacity awareness.

## Part 5: The Promise You Keep + The Lever You Keep

**To students:** Anyone who PIF'd Inner Circle has **unlimited credits forever**. We grandfather every existing student at the tier they paid for. Never revoke what was promised.

**To the business:** We add two dials admins can turn per-user:
1. `credits_unlimited: boolean` — overrides per-month cap
2. `max_tracked_sellers: number` — caps the token-heavy action

**Net effect:** Existing PIF students see no change. New SaaS users see defined tiers. Payment plan students feel the cap → motivated to finish plan → auto-unlock unlimited. Maliki-style abusers are bottlenecked by seller cap, not credit cap, so credits feel unlimited but actual Keepa spend stays bounded.

## Part 7: Cash Collection Playbook

**Payment processors:** Fanbasis + Whop only. NO Stripe. NO PayPal direct.
- Fanbasis = primary for coaching tiers (better high-ticket checkout flow, built-in order bumps)
- Whop = SaaS subscriptions + credit packs (existing integration, community gating)
- Wires accepted for $10K+ PIF (bypasses processor fees)

**Coaching sales:**
1. Coaching tiers sold via **Fanbasis** or **Whop** checkout links (Sabbo to provide existing links)
2. Wire accepted for PIF $10K+ — discount $200 if wire (saves processor fee)
3. Payment plans require card on file + authorization to auto-charge
4. Day-of-sale email: contract + access credentials + welcome call booked
5. Day 2 accountability call (Sales → CSM handoff)

**SaaS subscriptions:**
1. Whop subscriptions for monthly/annual tiers (existing plan_* slugs already in code for credit packs)
2. Failed card: auto-retry at day 1, 3, 7 → pause on day 10
3. Annual plans offered in-app on day 7, day 21, day 60 (convert monthly to annual)
4. Trial-to-paid sequence: 7 emails over 14 days + in-app reminders

**Refunds:**
1. SaaS: self-service through settings within 7 days
2. Coaching: email `support@` → routed to Sabbo → decided case-by-case
3. Chargebacks: automatic account freeze + ban on refund approval

## Part 10: Nova Chat Infrastructure — Claude Max, NOT Anthropic API

**Policy (Sabbo 2026-04-15):** Nova Chat and ALL AI features route through **Claude CLI** which uses the **$200/mo Claude Max plan**. We do NOT pay the Anthropic API per-token for any customer-facing feature.

**Why:** Flat $200/mo beats metered billing once usage crosses ~13M tokens/mo. Claude Max covers us. Anthropic API billing would erode margins on unlimited-Nova tiers.

**Rules:**
- All Nova Chat calls go through Claude CLI proxy (Flask JSON endpoint, non-streaming per existing rule)
- Product Analyzer summaries, Wholesale Research, Listing Optimizer — all route through CLI proxy
- NEVER import `@anthropic-ai/sdk` or `anthropic` Python SDK in customer-facing code paths
- The ONLY exception: if we hit Claude Max plan caps during peak hours, we can overflow to Anthropic API with explicit budget approval. Never default to it.

**Audit needed:** confirm `execution/discord_bot/`, `app/api/academy/chat/`, `app/api/product-analyzer/`, and `app/api/wholesale-research/` all use the Claude CLI proxy. If any still hit `api.anthropic.com` directly, migrate them. This should be logged in memory as a learned rule and enforced in PR reviews.
