AI 101

What Is AI, Really?

Let's cut through the noise. Here's what you need to know.

🧠

LLMs (Large Language Models)

ChatGPT, Claude, Gemini — AI models that understand and generate text. Think of them as extremely smart assistants that can read, write, reason, and follow instructions.

💬

Chat vs. Agents

Chat = you ask a question, it answers. Agents = you give it a task and it goes and does it. Opens websites, scrapes data, writes files, builds apps. Big difference.

Why This Matters for You

AI can scan hundreds of retailers, calculate profitability, build your brand website, write your ads, and answer your questions — all while you sleep.

THE FRAMEWORK

The DOE Framework

Directive — Orchestration — Execution. This is how you make AI actually reliable. Without this, AI makes mistakes that compound.

D

Directive

Plain English SOPs that tell the AI what to do. Step-by-step instructions, like you'd give a new employee. Written in Markdown files.

O

Orchestration

The AI brain. It reads the directive, makes decisions, picks the right tools, handles errors. This is Claude — the intelligent glue.

E

Execution

Deterministic scripts that do the actual work. Scraping, calculating, API calls. Reliable. Testable. Fast. No guessing.

WHY IT MATTERS

Why Can't AI Just Do Everything Itself?

Because errors compound. If AI is 90% accurate on each step...

90%
1 Step
73%
3 Steps
59%
5 Steps
35%
10 Steps

The fix: push the reliable work into deterministic scripts and let AI focus on decision-making. That's DOE.

BUILDING BLOCK #1

What Is CLAUDE.md?

It's your AI's operating manual. A file that loads automatically every time Claude Code starts. It tells the AI who you are, what your project is, and how to work.

# CLAUDE.md — Your AI's Instruction Manual

Project: My Amazon FBA Business
Stack: Python, Playwright, Keepa API
Rules:
  - Always check existing scripts before writing new ones
  - Products must have ROI > 30% to qualify
  - Never spend Keepa tokens without checking free sources first
  - Update the sourcing SOP when you learn something new

# Without this file, AI starts every session with ZERO context.
# With it, AI remembers everything — forever.
BUILDING BLOCK #2

What Are Directives?

Directives are SOPs (Standard Operating Procedures) written in Markdown. They tell the AI exactly what to do, step by step. The better your SOPs, the better your AI output.

# directives/amazon-sourcing-sop.md

Goal: Find profitable products to sell on Amazon FBA
Step 1: Scrape the retailer's clearance section
Step 2: For each product, search Amazon for a match
Step 3: Calculate FBA fees, shipping, and net profit
Step 4: Only keep products with ROI > 30%
Step 5: Check BSR — must be under 200,000
Step 6: Output a ranked list with buy links + margin

# The AI reads this before every sourcing run.
# When something breaks, update the SOP. System gets smarter.
BUILDING BLOCK #3

What Are Skills (Slash Commands)?

Skills are shortcuts that connect a trigger phrase to a directive and a script. Type one command, and the entire pipeline runs automatically.

# Skill file with YAML frontmatter:

---
name: source-products
description: AI-powered Amazon product sourcing
trigger: "find products", "source", "scan clearance"
tools: [Bash, Read, Write, Glob, Grep]
---

# Type "/source-products" and the AI:
# 1. Reads the sourcing SOP (directive)
# 2. Runs the sourcing script (execution)
# 3. Returns profitable products
# One command. Entire pipeline. Done.
KEY CONCEPT

Self-Annealing: The AI Gets Smarter

When something breaks, the AI doesn't just fix it — it updates its own instructions so the same mistake never happens again.

1

Something Breaks

Walmart changes their website layout. The clearance scraper returns empty results.

2

AI Fixes the Script

Detects the error, reads the new page, updates the scraping code, tests it, confirms it works.

3

Updates the SOP

Writes "Walmart layout changed 3/20 — new selectors" into the directive. Never breaks on this again.

Every error makes the system stronger. That's the DOE advantage.
