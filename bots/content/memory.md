# Content Bot — Memory
> bots/content/memory.md | Version 1.0

---

## Purpose

This file is your persistent memory. You read it on every heartbeat check and update it when Sabbo approves or rejects work. This is how you improve over time without losing context.

---

## Approved Work Log

When Sabbo approves a piece of content, log it here. This defines your quality bar.

Format:
```
### [DATE] — [ASSET TYPE] — [BUSINESS]
What it was: [1-line description]
Why it worked: [Sabbo's feedback or inferred reason]
Key element to repeat: [hook angle, structure, tone, format, etc.]
```

---

## Rejected Work Log

When Sabbo rejects or asks for a rewrite, log it here. This defines what to avoid.

Format:
```
### [DATE] — [ASSET TYPE] — [BUSINESS]
What failed: [1-line description]
Why it failed: [Sabbo's feedback]
What to do differently: [specific correction]
```

---

## Content Performance Log

Track which content types and topics perform best. Updated from analytics or Sabbo's feedback.

Format:
```
### [DATE] — [PLATFORM]
Content type: [VSL / short-form / carousel / blog / email]
Topic: [what the content was about]
Performance: [views, engagement, conversions — whatever is available]
Key takeaway: [what to repeat or avoid]
```

## Audience Insights

Learnings about what resonates with each business's audience.

Format:
```
### [BUSINESS — Agency / Coaching]
Insight: [what the audience responds to]
Evidence: [where this came from]
Implication: [what to create more of]
```
